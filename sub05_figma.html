MES 화면 설계서의 Figma 기반 디자인 시스템 전환

PPT 중심의 MES 화면 설계서를 Figma 디자인 시스템으로 전환해 설계–디자인–개발 연결을 강화하는 작업. 공통 컴포넌트화, 변수 관리, 간격/Auto Layout 규칙 수립을 진행하고, variant·네이밍·라이브러리·가이드로 확장 예정이다.

PPT 기반 설계서를 대체하는 Figma 디자인 시스템의 전체 구조를 한 화면에 정리했습니다.

Context

기존에는 PPT 기반으로 화면 설계서가 작성되었고, 이미 구축된 Figma 디자인은 개별 화면 단위로 존재했다. 이를 통합하여 설계–디자인–개발이 연결되는 구조를 목표로 디자인 시스템화 작업을 진행 중이다.

PPT 문서 기반에서 컴포넌트 조합 기반 설계서로 전환된 작업 방식을 비교했습니다. / 개별 화면을 해체해 공통 요소 중심의 컴포넌트 구조로 재정의했습니다.

Problem

  • PPT 기반 설계서로는 설계–디자인–개발 간 연결과 재사용 구조가 제한적이었다.
  • Figma 산출물이 화면 단위로 파편화되어 공통 컴포넌트와 규칙이 정리되지 않았다.
  • 시스템 확장을 고려한 간격, 상태, 네이밍, 사용 가이드 기준이 부재했다.

컬러와 타이포그래피를 변수로 통합 관리해 일관성의 기반을 마련했습니다.

Strategy

  • 화면 단위를 해체하고 버튼/인풋/카드 등 공통 요소 중심으로 컴포넌트화하여 재사용 가능한 Level 2 구조까지 정리한다.
  • 컬러와 타이포그래피를 변수로 관리해 일관성을 확보한다.
  • 핵심 컴포넌트에 spacing 기준을 부분 적용하고 Auto Layout 기반으로 구조를 안정화한다.
  • 상태별 대응을 위한 variant와 네이밍 규칙을 설계한다.
  • 단일 Design System 파일을 라이브러리로 분리·공유해 참조 기반 협업으로 전환한다.
  • 컴포넌트 조합 방식의 사용 가이드를 정의하고 Figma 설계서를 개발 단계와 직접 연결하는 체계를 마련한다.

간격 규칙과 Auto Layout을 적용해 구조 안정화를 진행하는 과정을 보여줍니다.

Execution

  • 버튼, 인풋, 카드 등 공통 요소를 식별해 컴포넌트화하고 재사용 가능한 Level 2 구성까지 완료했다.
  • 컬러·타이포그래피를 변수로 정리해 스타일을 통합 관리했다.
  • 핵심 컴포넌트에 간격 규칙을 부분 적용하고 Auto Layout 적용을 진행 중이다.
  • variant 설계와 네이밍 규칙 정리를 준비하고 있다.
  • Design System 파일 분리 및 라이브러리 퍼블리시 계획을 수립했다.
  • 팀 사용 가이드 구조와 예시를 정의해 컴포넌트 조합 중심의 작성 방식으로 전환을 준비하고 있다.

참조 기반 협업을 위해 라이브러리 퍼블리시와 사용 가이드를 함께 제공합니다.

Result

컴포넌트 기반 구조와 변수 관리 체계를 마련했고, 간격 규칙과 Auto Layout 적용을 시작했다. 라이브러리 공유와 사용 가이드 도입을 위한 다음 단계 준비가 진행 중이다.

insight

화면 단위 자산을 공통 컴포넌트와 변수 중심으로 재구성하면 설계서의 재사용성과 개발 연계 가능성이 높아진다. 간격 규칙과 Auto Layout은 확장성과 유지보수성의 핵심 기반이며, 참조 기반 라이브러리는 변경 관리에 유리하다.

Role 디자인 시스템 구조 설계, Figma 컴포넌트/스타일 정리, 규칙 수립 및 사용 가이드 기획  ·  Period 진행 중  ·  Type Design System 구축  ·  Output Figma 디자인 시스템 파일 및 컴포넌트 라이브러리(Level 2까지), 컬러/타이포 변수, 초기 spacing 규칙과 Auto Layout 구조, variant·네이밍·사용 가이드 초안(계획)
